## üîÑ Making Changes

When updating configuration:
1. **Edit the root file** (the one tools actually use)
2. **Copy to this folder** for reference: `cp <filename> config/`
3. Or edit both if you want to keep them in sync

## üìù Why This Structure?

Many build tools and frameworks expect configuration files at the project root. This folder serves as:
- **Documentation** - See all configs in one place
- **Backup** - Reference copies if needed
- **Organization** - Cleaner root directory view

The actual working configs remain at root where tools expect them.
